# 菜谱信息架构优化方案

## 📊 当前信息架构分析

### 当前布局结构
```
┌─────────────────────────────────┐
│ [图片]                          │
├─────────────────────────────────┤
│ 标题                            │
│ 描述                            │
│ [时间] [份量] [难度]            │
├─────────────────────────────────┤
│ 烹饪步骤                        │
│ 1. ...                          │
│ 2. ...                          │
├─────────────────────────────────┤
│ 🔥 融合模式标识                 │  ← 问题：在底部
│ ✨ AI膳食优化建议               │  ← 问题：在底部
├─────────────────────────────────┤
│ [保存] [分享]                   │
└─────────────────────────────────┘
```

### 问题分析

#### 1. **信息层级混乱**
- 融合模式标识：说明"为什么是这个版本"（决策依据）
- AI膳食优化：说明"如何针对你优化"（个性化价值）
- 两者都在底部，用户需要滚动才能看到

#### 2. **阅读流程不合理**
- 用户期望：标题 → 为什么推荐 → 怎么做 → 操作
- 当前流程：标题 → 怎么做 → 为什么推荐 → 操作
- **问题**：用户已经看完步骤，才看到推荐理由

#### 3. **视觉权重不当**
- 融合模式和AI优化是**核心价值点**，应该在视觉上更突出
- 当前放在底部，容易被忽略

#### 4. **信息分组不清晰**
- 融合模式 = 外部热点数据（社交证明）
- AI优化 = 内部个性化（健康匹配）
- 两者性质不同，应该分开展示

## 🎯 优化方案

### 方案A：分层展示（推荐）⭐

#### 信息架构重组
```
┌─────────────────────────────────┐
│ [图片]                          │
├─────────────────────────────────┤
│ 标题                            │
│ 🔥 [融合模式徽章]              │  ← 顶部：决策依据
│ 描述                            │
│ [时间] [份量] [难度]            │
├─────────────────────────────────┤
│ 烹饪步骤                        │
│ 1. ...                          │
│ 2. ...                          │
├─────────────────────────────────┤
│ ✨ AI膳食优化建议               │  ← 中部：个性化价值
│ (已关联档案)                    │
├─────────────────────────────────┤
│ [保存] [分享]                   │
└─────────────────────────────────┘
```

#### 优势
- ✅ 信息层级清晰：决策依据在顶部，个性化价值在中部
- ✅ 阅读流程合理：先看为什么，再看怎么做
- ✅ 视觉权重正确：重要信息前置

## 💡 推荐方案：方案A（分层展示）

### 实施细节

#### 1. **融合模式标识 - 顶部展示**
位置：标题下方，描述上方
样式：徽章式设计，不占用太多空间
内容：
- 图标 + "热点融合"
- 平台数据（TikTok 500万+）
- 营养优化数据（热量↓12%）

#### 2. **AI膳食优化建议 - 步骤后展示**
位置：烹饪步骤之后，操作按钮之前
样式：卡片式设计，与步骤区分
内容：
- 标题："✨ AI膳食优化建议"
- 说明：针对您的健康需求优化

#### 3. **视觉层次**
- 融合模式：徽章式，轻量级，不打断阅读
- AI优化：卡片式，重量级，强调个性化价值

## 📐 信息架构原则

### 1. **F型阅读模式**
- 用户视线：左上 → 右上 → 左下 → 右下
- 重要信息放在左上和左侧

### 2. **信息优先级**
1. **P0 - 核心内容**：标题、图片、步骤
2. **P1 - 决策依据**：融合模式（为什么是这个版本）
3. **P2 - 附加价值**：AI优化（个性化价值）
4. **P3 - 操作**：保存、分享

### 3. **信息分组**
- **外部数据**：融合模式（社交证明、热点数据）
- **内部优化**：AI膳食优化（个性化匹配）
- **操作区**：保存、分享

## 🎨 UI/UX 优化建议

### 融合模式徽章设计
```
┌─────────────────────────────────┐
│ 🔥 热点融合                     │
│ 📱 TikTok 500万+ | ⬇️ 热量-12%  │
└─────────────────────────────────┘
```
- 位置：标题下方，描述上方
- 样式：横向徽章，不占用垂直空间
- 交互：可点击查看详情

### AI优化卡片设计
```
┌─────────────────────────────────┐
│ ✨ AI膳食优化建议               │
│ ─────────────────────────────── │
│ 针对您的健康需求，AI已自动优化   │
│ 烹饪方案...                      │
└─────────────────────────────────┘
```
- 位置：步骤之后，按钮之前
- 样式：独立卡片，视觉突出
- 交互：可折叠/展开

### 桌面端
- 融合模式：可考虑侧边栏固定
- AI优化：保持卡片式，但可更宽
